序章

俗话说，万事开头难。但对我来说，我感觉阅读学习计算机类书籍来说，反而是开头容易。为什么呢？计算机类的书籍一般都是开头简单，但到后面就会变得愈发地难。虽说是循序渐进地，但自己有时候在开头了之后，学着学着发现看不到应用场景，也就是说看了，也不知道拿来干啥，就感觉学得挺空的，同时在短时间内还看不到自己的进步，就慢慢地停止了学习。这还是自己的问题，是由于自己的内心过于急躁所导致的。而现在呢，自己多少平静了一些，也能按自己的步速来，另一方面，我现在也知道学这个东西是干什么的了，而且在学的同时还会写代码去练习，这跟以往空学是不一样的。愿我能通过阅读这本书籍，去回忆熟悉起我本该掌握的东西。

关键就还是记住这句话：指针p的内部存储对象地址，而*p表示求指针内部地址对应对象的值。操作p是操作“指针对象”p本身，而操作*p却是操作p所指向的对象。当碰到一时间难以理解的指针操作，照着这句话去思考理解即可，因为语言和代码本身都是人创造的，它必然是有着人可以理解的逻辑方式，只是可能需要稍微花点时间理一下。

注：一个未指向任何对象的指针，其地址值为0。我们需要在初始化指针时，为其赋值为0，避免其指向一个不该指向的地址，从而避免出错。其实，在初始化变量和数组的时候，也应该这么做，避免其初始值变得非常乱，使得代码难以调试。但很多代码并没有这么做，不清楚是为什么（这一说法仅针对类C的代码，C++的一些容器类对象应该是有构造函数帮忙做类似的事情）。

What is the difference between g++ and gcc?

For c++ you should use g++.

It’s the same compiler (e.g. the GNU compiler collection). GCC or G++ just choose a different front-end with different default options.

In a nutshell: if you use g++ the frontend will tell the linker that you may want to link with the C++ standard libraries. The gcc frontend won’t do that (also it could link with them if you pass the right command line options).

练习1.2

题目：将string头文件注释掉，重新编译这个程序看看会发生什么事。然后取消对string头文件的注释，再将下一行注释(// using namespace std;)，又会发生什么事情？

回答：将string头文件注释掉，会发现程序仍然可以正常编译，系统版本macOS Monterey 12.2.1。按常理来说，应该是编译不通过的，因为你后面都用了string类型的变量，但你却没有包含这个头文件。于是，在这里找到答案。也就是说，在c++中，包含任何一个标准头文件相当于包含任何或所有其他标准头文件。因此，如果你包含<iostream>，那就相当于也包含<algorithm>， <vector>， <string>，等等。我不清楚书的作者是想把我们引导到何处，但实际操作却是这样的，也算是增加了一些小知识吧。但实际操作时，是应该把所以需要的库去包含在内，以方便迁移。因为上面的特性并非所以的编译器都支持，你也不知道自己的代码将在何处被执行。

将using namespace std;注释掉，编译不通过，提示之一：unknown type name 'string'; did you mean 'std::string'。因为没有声明std这个命名空间，来让编译器识别这个类型。
